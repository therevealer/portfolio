Process

01

Understand the product

I read the codebase, run the API, and talk to engineers before writing a single word. Documentation starts with comprehension.

02

Audit what exists

Gaps, inconsistencies, outdated content. I map the full documentation surface before adding to it.

03

Structure before writing

I apply the Diátaxis framework, tutorials, how-tos, references, and explanations each serve a distinct reader goal.

04

Write for the developer

Precise, scannable, and tested. Every code sample runs. Every parameter is documented. Nothing is assumed.

05

Build the system

Docs-as-code pipelines, linting rules, CI/CD checks. I set up infrastructure so quality scales with the team.
